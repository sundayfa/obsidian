На MacOS, Win - Docker Desktop создаёт виртуалку Linux для запуска dockerd - службы docker.
На Linux:
- ``` sudo systemctl start docker``` - включение демона docker

- `sudo usermod -aG docker *username*` - добавление прав пользователю, чтобы не писать sudo

- `newgrp docker` - перезапуск

- `docker version`

- `docker ps -a` - просмотр всех контейнеров, без `-a` только запущенные

- `docker images` - просмотр всех образов

- `docker run *имя_образа*` - создаёт и запускает контейнер из указанного образа, а также скачивает его с [Docker Hub](https://hub.docker.com), если образа нет локально
> -d - запускает контейнер в background'е
> --name - задаёт имя
> -p внешний_порт:порт_контейнера - проброс портов
> -v \*абсолютный\_путь\_до\_файла:путь\_до\_папки\_в\_контейнере\_где\_будет_файл\* *имя_образа* - добавить файл с машины в контейнер
> --rm - удаление контейнера после его остановки
> docker run -d -p 8080:80 --name my_nginx nginx - запуск образа nginx

- `docker rm *id_контейнера* или *name_контейнера*` - удалить контейнер

- `docker container prune` - удалить все остановленные контейнеры

- `docker container inspect *id_контейнера* или *name_контейнера*` - информация о контейнере

- `docker stop *id_контейнера* или *name_контейнера*`  - остановить запущенный контейнер

- `docker kill *id_контейнера* или *name_контейнера*` - убить процесс внутри запущенного контейнера

- `docker exec -it *id_контейнера* или *name_контейнера* *название_процесса*` - запуск дополнительного процесса в уже запущенном контейнере
> например, bash позволяет зайти в оболочку bash внутри контейнера

- `hostname -i` - ip-адрес контейнера

Без активного процесса в контейнере он будет остановлен. Некоторые процессы требуют опции `-it` (интерактивный терминал), чтбы взаимодействовать с ними.

Пример Dockerfile:
```
FROM имя_базового_образа:тэг_базового_образа

WORKDIR /имя_рабочей_директории_в_образе

COPY . . - копирование всех файлов с локальной текущей папки машины в папку WORKDIR

CMD ["python", "main.py"] - будет выполнена команда python с аргументом main.py
```

Команды для создания образа:
- `docker build .` - запуск создания образа

- `docker build -t my-name:tag-name.` - добавление имя образа и тэга
