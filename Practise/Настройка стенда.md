Хост - Fedora 43.

Сначала установил виртуалки: Ubuntu, Debian и Windows 11.

На Ubuntu установил Wazuh: Indexer, Server и Dashboard. 

На Debian и Windows 11 агенты с помощью интерфейса Wazuh.

Чтобы виртуалки были доступны с внешней сети, добавить конфиг на хосте:
```
sudo tee /etc/nftables.conf > /dev/null <<EOF #!/usr/sbin/nft -f flush ruleset table inet filter { chain input { type filter hook input priority 0; policy accept; } chain forward { type filter hook forward priority 0; policy drop; # 1. Allow established/related traffic (CRITICAL for internet) ct state established,related accept # 2. Allow local network (enp4s0) to reach VMs (virbr0) iifname "enp4s0" oifname "virbr0" ip daddr [192.168.122.0/24](https://192.168.122.0/24) accept # 3. Allow VMs (virbr0) to reach the internet (enp4s0) iifname "virbr0" oifname "enp4s0" accept # 4. Allow VMs to talk to each other iifname "virbr0" oifname "virbr0" accept } chain output { type filter hook output priority 0; policy accept; } } # THIS SECTION WAS MISSING - IT ENABLES INTERNET (NAT) table ip nat { chain postrouting { type nat hook postrouting priority 100; policy accept; ip saddr [192.168.122.0/24](https://192.168.122.0/24) oifname "enp4s0" masquerade } } EOF
```

Также на Windows виртуалке нужно сделать:
```
netsh advfirewall firewall add rule name="Allow Ping" dir=in action=allow protocol=icmpv4 powershell Get-NetConnectionProfile | Set-NetConnectionProfile -NetworkCategory Private
```
